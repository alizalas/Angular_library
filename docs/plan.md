# План проекта: Личная библиотека (BookShelf)

## Цель проекта
Создать веб-приложение для каталогизации личной библиотеки с возможностью отслеживания прочитанных книг, добавления заметок и цитат, а также обмена рекомендациями.

## Краткое описание продукта
BookShelf - это персональный книжный менеджер, который помогает пользователям:
- Вести учет своих книг
- Отслеживать прогресс чтения
- Сохранять важные цитаты и мысли
- Делиться рекомендациями с друзьями
- Анализировать свои читательские привычки

## Пользовательские сценарии

### Сценарий 1: Добавление и отслеживание книги
1. Пользователь авторизуется в системе
2. Находит книгу через поиск или добавляет вручную
3. Указывает метаданные (автор, год, издательство, жанр)
4. Устанавливает статус "Планирую прочитать"
5. Позже меняет статус на "Читаю", затем "Прочитано"
6. После прочтения ставит оценку и пишет рецензию

### Сценарий 2: Работа с заметками и цитатами
1. Пользователь открывает книгу со статусом "Читаю"
2. Добавляет заметки по ходу чтения
3. Сохраняет понравившиеся цитаты с указанием страницы
4. Просматривает все сохраненные цитаты в разделе "Мои цитаты"

### Сценарий 3: Создание рекомендательного списка
1. Пользователь переходит в раздел "Рекомендации"
2. Создает новый список (например, "Лучшее фэнтези 2024")
3. Добавляет прочитанные книги с высокими оценками
4. Делится ссылкой на список с друзьями
5. Друзья могут просматривать список без авторизации

### Сценарий 4: Анализ статистики чтения
1. Пользователь открывает дашборд
2. Видит количество прочитанных книг за месяц/год
3. Просматривает распределение по жанрам
4. Анализирует средние оценки по категориям
5. Отслеживает прогресс чтения

## Риски

| Риск | Вероятность | Влияние | Решение |
|------|-------------|---------|---------|
| Сложность с Taiga UI 4 в Angular 21 | Средняя | Высокое | Раннее прототипирование, проверка совместимости |
| Проблемы с mock-сервером | Низкая | Среднее | Использование json-server с кастомными middleware |
| Производительность при большом количестве книг | Средняя | Среднее | Виртуальный скроллинг, пагинация на стороне сервера |
| Сложность с Signal Store | Высокая | Среднее | Документация и примеры из официальных источников |

## Технический план

### 1: Базовая архитектура (Коммит 2)
- Настройка Angular 21 проекта
- Установка Taiga UI 4
- Базовая структура папок (core, shared, features)
- Настройка маршрутизации с lazy loading

### 2: Авторизация (Коммит 3)
- Сервис авторизации
- Guards для защищенных маршрутов
- Interceptors для токенов
- Mock API для auth endpoints

### 3: Управление книгами (Коммит 4)
- CRUD операции для книг
- Форма с валидацией
- Поиск и фильтрация
- Signal Store для состояния книг

### 4: Статусы и статистика (Коммит 5)
- Управление статусами чтения
- Статистика и прогресс
- Дашборд с аналитикой

### 5: Заметки и цитаты (Коммит 6)
- CRUD для заметок
- Сохранение цитат
- Привязка к книгам

### 6: Рекомендации и социализация (Коммит 7)
- Создание рекомендательных списков
- Публичные ссылки
- Импорт/экспорт данных

### 7: Тестирование и оптимизация (Коммит 8)
- Unit тесты (Jest)
- Component тесты (Playwright)
- Lighthouse оптимизация
- CI/CD настройка
